The AJAX Bridge: Fetch API

Connecting the View (HTML) to the Controller (PHP) using modern JavaScript.

Why Fetch?

The Fetch API allows us to send and receive data from the server **asynchronously**—meaning the user experience is fast, the page doesn't reload, and the Controller can return simple, structured data (like JSON) instead of a whole new HTML page.

1. The View (HTML Form)

The user interface that captures the data.

2. The Bridge (JavaScript Fetch)

The script that intercepts the submission and performs the asynchronous call. 

const form = document.getElementById('registrationForm');
const messageBox = document.getElementById('messageBox');
const controllerUrl = '/register'; // This maps to your UserController

form.addEventListener('submit', async (event) => {
    // 1. CRITICAL: Prevent the default browser form submission (page reload)
    event.preventDefault(); 
    
    // Clear previous messages
    messageBox.classList.add('d-none');
    
    // 2. Prepare Data: FormData handles field names/values automatically
    const formData = new FormData(form);

    // 3. Initiate the Fetch Request (The Magic)
    try {
        const response = await fetch(controllerUrl, {
            method: 'POST', // Use POST for registration/creation
            // Note: Content-Type is NOT needed here if sending FormData
            body: formData // The data object created above
        });

        // 4. Handle HTTP Errors (e.g., 404, 500)
        if (!response.ok) {
            // Throwing an error sends control to the catch block
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        // 5. Read the Response: Controller MUST return JSON
        const result = await response.json(); 

        // 6. Handle Controller Response (Business Logic Errors/Success)
        if (result.status === 'success') {
            // Success: Update the View with a positive message
            messageBox.textContent = result.message;
            messageBox.className = 'mt-4 p-3 rounded alert-success';
            form.reset(); // Clear the form on success
        } else {
            // Failure: Update the View with the error message
            messageBox.textContent = 'Registration failed: ' + result.message;
            messageBox.className = 'mt-4 p-3 rounded alert-danger';
        }
    } catch (error) {
        // 7. Network/Code Errors
        console.error('Fetch Error:', error);
        messageBox.textContent = 'An unexpected error occurred. Please try again.';
        messageBox.className = 'mt-4 p-3 rounded alert-warning';
    } finally {
        messageBox.classList.remove('d-none'); // Show message box
    }
});

3. The Controller's Response (PHP)

What your PHP Controller must send back for the JavaScript to read.

Example PHP Controller Snippet (After logic runs):

<?php
header('Content-Type: application/json'); // CRITICAL header!

if ($registration_successful) {
    // Return Success JSON
    echo json_encode([
        'status' => 'success',
        'message' => 'Registration successful! Check email for verification.'
    ]);
} else {
    // Return Error JSON
    echo json_encode([
        'status' => 'error',
        'message' => $validation_error_message // e.g., "Email already in use"
    ]);
}
exit; // Stop execution after sending JSON
?>

4. The Asynchronous Flow

This is the asynchronous data path, which makes your application fast and modern.

When the user clicks "Register," the **Fetch API** sends the data to the Controller, but the browser stays put. The Controller does the heavy work (validation, PDO/database interaction), and sends back a small package of **JSON** data. JavaScript then instantly updates the View based on that package.
