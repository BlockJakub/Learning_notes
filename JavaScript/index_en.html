Understanding Bracket Notation in JavaScript

Bracket notation in JavaScript is a way to access object properties and array elements using square brackets []. It is not just syntactic decoration, but a practical tool designed to handle situations where dot notation is not sufficient.

Think of an object as a cabinet with many compartments. Dot notation works only when you know the exact name of the compartment in advance. Bracket notation allows you to determine the compartment name at runtime — from variables, expressions, or dynamically created strings.

Basic Example

const user = {
    name: "Jacob",
    age: 29
};

user.name;        // dot notation
user["name"];     // bracket notation

At first glance, both approaches look identical. The real difference appears when property names are dynamic.

1) Dynamic Property Names (Main Use Case)

const key = "age";

user[key];   // 29
user.key;    // undefined

Dot notation does not evaluate variables — it treats key literally as a property name. Bracket notation evaluates the variable and uses its value. This behavior is essential when working with forms, JSON data, APIs, and databases.

2) Invalid Property Names for Dot Notation

const product = {
    "price-usd": 12,
    "in stock": true
};

product["price-usd"];
product["in stock"];

Dot notation fails when property names contain hyphens, spaces, or start with numbers. Bracket notation has no such limitations.

3) Expressions and Computed Property Names

const settings = {
    themeDark: true,
    themeLight: false
};

const mode = "Dark";
settings["theme" + mode]; // true

Dot notation cannot build property names dynamically. Bracket notation can construct property names using expressions.

4) Required for Arrays

const colors = ["red", "green", "blue"];

colors[0];     // red
colors["0"];   // red

Arrays are technically objects in JavaScript. Their indexes are object keys, which makes bracket notation mandatory. Without it, arrays would not function.

5) Adding, Deleting, and Checking Properties

user["email"] = "test@mail.com";
delete user["age"];

"email" in user; // true

Bracket notation enables flexible property manipulation and existence checks when working with dynamic data.

When to Use Which Notation

Dot notation should be used when the property name is known in advance and is a valid identifier. It is shorter and more readable.

Bracket notation should be used when property names are stored in variables, built at runtime, contain special characters, or when working with JSON, APIs, databases, or arrays.

Summary: Dot notation is convenience. Bracket notation is power. As soon as you start building real-world applications, you will rely on bracket notation constantly — often without even noticing. JavaScript is a dynamic language, and bracket notation is one of its sharpest tools.
